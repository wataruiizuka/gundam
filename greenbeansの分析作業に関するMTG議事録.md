# GreenBeans分析作業 議事録

## 1. 会議の概要と全体方針

**目的:** 6月1日に配信した広告の効果検証と、次回7月の配信に向けたターゲット選定のための分析タスクの共有、およびメンバー間の役割分担の決定です。

**背景:** GreenBeans（イオンのネットスーパー）は1都3県（東京・千葉・埼玉・神奈川）でサービスを展開しています。今回は新規会員獲得を目的に広告を配信しました。現在、作業量が過剰気味であるため、メンバー間でタスクを分担して進めます。

**これまでの取り組み:** 既存会員を「カスタマーDNA」を用いてクラスター分析（5つのクラスター）し、1都3県の非会員（未利用者）を同じクラスターに当てはめて、クラスターごとにクリエイティブを変えて広告を配信しました。6月1日〜7日の配信結果から、すでにクリック率（CTR）の集計は完了しています。

**今後のゴール:** 分析の目的と内容を全員が理解し、スケジュールと作業分担を確定させることです。分析結果をもとに7月の配信ターゲットの精度を高め、GreenBeans側（先方の増田さん、岡崎さんなど）へ有効性を示して契約獲得・維持に繋げます。

## 2. メンバーごとの作業内容（役割分担）

### タナカさんの担当タスク

**担当領域:** 6月配信の広告接触者に関する分析（ナンバー1タスク）を行います。

**目的:** 広告接触者と非接触者の定性的特徴を明らかにし、どのような顧客が広告に反応しやすかったのかを可視化します。

**作業内容:** カスタマーDNAの属性別に、広告接触者と非接触者の比率を比較するクロス集計やインデックス比較（レベル2の「標準化差分やオッズ比を用いた分析」を目標）を行います。可能であれば、5つのクラスター内それぞれでこの比較分析を実施します。

**注意点:** 予測モデル（決定木）の作成はデータボリューム（会員数が300万弱）が大きく、Databricksの負荷や作業期間（約1週間）を考慮すると時間がかかりすぎるため、今回はレベル2の分析に絞って対応します。また、非接触者のカスタマーDNAが紐付かない可能性もあるため、事前にデータのボリューム感を確認しながら進めます。

### イイヅカさんの担当タスク

**担当領域:** 7月配信に向けた配信対象者の分析およびリスト作成のうち、「離反防止分析（ナンバー3タスク）」を担当します。

**目的:** 過去に購入履歴のある会員の離反を防ぎ、継続利用（リテンション）を向上させるための最適なアプローチを導き出します。

**作業内容:** 直近半月分（昨年10月〜今年3月）の購買データを用い、既存の離反予測プログラムをGreenBeans用にカスタマイズ・横展開して、離反確率のスコアリングとクラス分けを行います。

**注意点:** 4月・5月の最新データがまだ揃っていないため、今回は3月までのデータ（過去データ）を使って2ヶ月後の離反を予測する形（実質的な検証・モデリング）で進めます。データの期間やスコアリングの判定条件（パラメータ）は、GreenBeansの特性（購買間隔の平均が20〜30日程度であることなど）に合わせてイイヅカさん側で調整・判断して進めます。

### コサカさんの担当タスク

**担当領域:** 全体の進行管理と、7月配信に向けた「1回のみ購買者に対する3回目までのステップアップ分析（ナンバー2タスク）」を担当します。

**目的:** 1回しか購入していない顧客のアップセルを促し、3回目までの購買に繋げて利用を定着させます。

**作業内容:** 購買データの中で「購買回数が1回のみ」の顧客（初回購買日と最終購買日が一致している顧客）を抽出し、複数回購入している顧客との特徴の差異を分析します。カスタマーDNAや属性情報からクラスター分析を行い、複数回購買者に近い特徴を持つ「2回目以降の購買確率が高そうな顧客」を特定して配信リスト（レベル3〜4を視野）を作成します。

## 3. スケジュールとネクストアクション

**来週〜6月19日（金）:** 各担当者は、担当する分析作業および報告資料の作成を並行して進めます。

**6月23日（火）:** 内部での進捗共有ミーティングを設定し、イイヅカさんの離反防止分析の状況や、コサカさんのステップアップ分析の状況をすり合わせます。

**6月24日（水）〜26日（金）:** 内部フィードバックをもとに、資料の修正やブラッシュアップを行います。

**6月末（6月29日・30日頃目安）:** 先方（GreenBeans側）とのミーティングを行い、分析結果の共有と今後の方向性について合意を取ります。

**7月10日前後（目安）:** 分析結果から抽出された「配信対象者IDリスト（アライオンメンバーID）」をもとに、7月の広告配信（広告の出し分け）を実施します。

## 4. 決定事項および確認事項

**使用データについて:** 使用する4つの主要テーブル（広告配信対象者、広告接触者、購買履歴、会員マスタ）はすべてDatabricks上（一部はコサカさんのスキーマ内）に用意されており、メンバー全員に閲覧権限が付与されています。

**売上の集計基準:** 購買履歴（パチャスログ）における売上金額の集計は、「値引き前（値引き前金額）」を基準として処理することで合意しました。

**アライオンメンバーIDの扱い:** データ内の顧客識別子は「アライオンメンバーID」で統一されており、そのまま紐付けに使用できます（一部テーブルにある「クラブ組織会員番号」は今回は使用しません）。

**中間テーブルの作成環境:** 分析の過程でブロンズ・シルバー層などの中間テーブルを作成する際、コサカさんのスキーマ内に作成できない場合は、ヒグチさんにお願いして各メンバーの個人スキーマを作成してもらい、そこで個別にテーブルを作成して進めます。

**タスク管理:** 今回の分担に沿って、コサカさん側でJiraチケットを起票し、各担当者（タナカさん、イイヅカさん）にアサインを行います。
